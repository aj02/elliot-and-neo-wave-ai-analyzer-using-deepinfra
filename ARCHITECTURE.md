# Architecture

> **Disclaimer:** wave-agent is an educational and engineering demo. It is not investment advice. The system proposes structural interpretations and deterministic levels; humans decide actions.

This document explains the deterministic / LLM split — the intellectual core of the project — and the dataflow from CSV upload to final report.

## Design principles

1. **Determinism first.** Anything that can be computed deterministically *is* computed deterministically, in Python. This includes pivots, swings, channel fits, Fibonacci levels, time ratios, invalidation prices, and rule compliance.
2. **LLMs interpret structured inputs.** LLM agents never see raw OHLCV. They see a compact `StructureSummary` (~80–150 tokens per timeframe) with numeric properties pre-computed.
3. **LLMs produce schema-bound JSON.** No prose, no chain-of-thought escapes the agent. Internal reasoning is discarded; only the schema-validated output reaches the rest of the system.
4. **Rules gate LLM output.** Every LLM-proposed count is checked by deterministic Python validators (Prechter for Elliott, Neely for NEOWave). Rule-violating counts are rejected before they reach the user.
5. **Numbers come from Python.** Invalidation levels, Fibonacci targets, and channel projections rendered next to LLM rationale are computed in Python — never generated by the LLM. LLMs hallucinate numbers; the geometry of the data does not.
6. **Cache aggressively.** Same input hash + same agent + same model = cached result with a 7-day TTL.

## Dataflow

```
            POST /upload (multipart, N CSVs + per-file timeframe label)
                                  │
                                  ▼
                  CSV loader + validators (strict)
                                  │
                                  ▼
                       UploadedTimeframe[]
                                  │
                                  ▼
            POST /analyze { session_id, instrument_name }
                                  │
                                  ▼
                       Orchestrator (asyncio)
                                  │
            ┌─────────────────────┼─────────────────────┐
            ▼                                           ▼
   For each timeframe:                                 (parallel
       pivots → swings → channels → fibonacci          per timeframe)
                            │
                            ▼
                  StructureSummary (~80–150 tokens)
                            │
            ┌───────────────┴───────────────┐
            ▼                               ▼
      Elliott Agent                   NEOWave Agent
        (Haiku)                          (Haiku)
            │                               │
            └─────────►  Validator  ◄───────┘
                  (deterministic Python rules)
                            │
                            ▼

                            │

                  Synthesis Agent (Sonnet, single call)
                            │
                            ▼
                     AnalysisReport
                            │
                            ▼
                Postgres (run storage) + WebSocket stream
```

## Components

### Preprocessing (`backend/app/preprocessing/`)

| Module          | Responsibility                                                                  |
| --------------- | ------------------------------------------------------------------------------- |
| `csv_loader.py` | Strict CSV parsing with detailed, user-actionable error messages.               |
| `validators.py` | Schema, monotonicity, gap detection, OHLC sanity, row-count guards.             |
| `pivots.py`     | Custom ZigZag pivot detection (configurable pct threshold).                     |
| `swings.py`     | Labels pivots HH/HL/LH/LL, computes swing pct/bars/retracement.                 |
| `channels.py`   | Fits parallel channels through pivot sequences (regression-based).              |
| `fibonacci.py`  | Retracement, extension, projection, time ratios — pure math.                    |
| `structure.py`  | Assembles the compact `StructureSummary` that LLMs consume.                     |

### Rules (`backend/app/rules/`)

These modules are **deterministic Python validators**. They are run *after* the LLM proposes a count.

| Module             | Encodes                                                                         |
| ------------------ | ------------------------------------------------------------------------------- |
| `elliott_rules.py` | Wave 2 retracement < 100% of wave 1; wave 3 not the shortest; wave 4 doesn't enter wave 1 territory in non-diagonals; alternation; channeling. |
| `neowave_rules.py` | Rule of Similarity & Balance, channeling, time relationships, monotony, complexity. |

See [ELLIOTT_RULES.md](ELLIOTT_RULES.md) for the full rule list with citations.

### Agents (`backend/app/agents/`)

Built on PydanticAI. Schema-bound output, no tools (the agents do pure reasoning over structured input).

| Agent              | Model           | Calls per run             | Output schema                         |
| ------------------ | --------------- | ------------------------- | ------------------------------------- |
| Elliott Agent      | Claude Haiku    | 1 per timeframe           | `list[ElliottCount]`                  |
| NEOWave Agent      | Claude Haiku    | 1 per timeframe           | `list[NeowaveCount]`                  |
| Synthesis Agent    | Claude Sonnet   | 1 total                   | `SynthesisReport` (3 ranked scenarios) |

The orchestrator runs Elliott + NEOWave **in parallel per timeframe** via `asyncio.gather`, passes results through the Validator, then makes a single Synthesis call.

### Caching (`backend/app/services/cache.py`)

- Key: `SHA-256(StructureSummary JSON + agent name + model name)`
- Backend: Redis
- TTL: 7 days

Cache is content-addressed. Once a structure is computed for a given dataset, agent outputs for it are reusable indefinitely (within TTL).

### WebSocket events (`backend/app/agents/events.py`)

The `/ws/runs/{run_id}` endpoint streams a typed event sequence:

```
preprocessing_started → preprocessing_completed →
  agent_started (× N) → agent_completed (× N) →
  validation_completed →
  synthesis_started → synthesis_completed →
  run_completed
```

Errors are emitted as `error` events and terminate the run.

## What the LLM never does

- Compute Fibonacci numbers, invalidation prices, channel projections, or any other numeric output.
- See raw OHLCV.
- Decide whether a count complies with Elliott or NEOWave rules — the deterministic Validator decides that.
- Output prose; only schema-bound JSON is permitted.
- Use words like "buy", "sell", "predict", "target price", "high probability". The system prompt forbids them and the schema doesn't accept them.

## What the LLM does

- Propose candidate wave counts and rank them by structural fit.
- Identify cross-timeframe alignment ("the daily wave 4 retracement aligns with the weekly wave 2 of an emerging impulse").
- Identify cross-framework agreement between Elliott and NEOWave interpretations.
- Provide one-sentence rationales that reference specific pivot indices and pre-computed numeric levels.
